MxTac Documentation

Installation, Configuration & Operations Manual — v2.0

MxTac is an open-source security integration platform that unifies best-of-breed OSS security tools under a single MITRE ATT&CK-native interface. It normalizes events via OCSF, executes Sigma rules natively, and provides cross-tool attack chain detection.

  +------------------+     +------------------+     +------------------+
  |    Wazuh 4.7+    |     |    Zeek 6.x      |     |  Suricata 7.x   |
  +--------+---------+     +--------+---------+     +--------+---------+
           |                         |                         |
           +------------+------------+------------+------------+
                        |       Connectors        |
                        v                         v
               +--------+---------+      +--------+---------+
               |  OCSF Normalizer |      |  Kafka / Redpanda |
               +--------+---------+      +--------+---------+
                        |                         |

               +------------------+      +------------------+

Quick Start

Get MxTac running in under 5 minutes with Docker Compose.

1

Clone the repository

git clone https://github.com/fankh/new-research.git
cd new-research/mitre-attack/mxtac/app
2

Configure environment

cp backend/.env.example backend/.env
# Generate a secret key
sed -i "s/CHANGE_ME_GENERATE_WITH_OPENSSL_RAND_HEX_32/$(openssl rand -hex 32)/" backend/.env
3

Start all services

docker compose up -d

This starts: Backend API (port 8080), Frontend (port 5173), PostgreSQL (5432), Valkey (6379), and OpenSearch (9200).

4

Access MxTac

Dashboardhttp://localhost:5173
APIhttp://localhost:8080
API Docshttp://localhost:8080/docs
5

Login

# Default development credentials
Email:    analyst@mxtac.local
Password: mxtac2026
Warning
Change the default credentials and SECRET_KEY before deploying to production.

Installation

Docker Compose (Development)

The development setup includes hot-reload for both backend and frontend, with source code mounted as volumes.

# Start core services (backend, frontend, postgres, valkey, opensearch)
docker compose up -d

# Optional: Start with OpenSearch Dashboards
docker compose --profile dashboards up -d

# Optional: Start with Kafka (Redpanda)
docker compose --profile kafka up -d

# Optional: Start everything
docker compose --profile dashboards --profile kafka up -d

Docker Compose (Production)

The production setup uses pre-built images, Nginx reverse proxy, and secure defaults.

1

Create production environment file

# Create .env for production
cat > .env <<'EOF'
DOMAIN=mxtac.example.com
SECRET_KEY=$(openssl rand -hex 32)
POSTGRES_PASSWORD=$(openssl rand -base64 24)
EOF
2

Prepare TLS certificates

mkdir -p nginx/ssl
# Copy your certificates
cp /path/to/fullchain.pem nginx/ssl/
cp /path/to/privkey.pem nginx/ssl/
3

Deploy

docker compose -f docker-compose.prod.yml up -d
Security Checklist (Production)
  • Replace SECRET_KEY with a strong random value
  • Set a strong POSTGRES_PASSWORD
  • Enable TLS/HTTPS via Nginx
  • Set DEBUG=false
  • Restrict CORS_ORIGINS to your domain only
  • Enable OpenSearch security plugin
  • Change default user credentials on first login

Manual Setup

For environments where Docker is not available. Requires Python 3.12+, Node.js 20+, PostgreSQL 16, and OpenSearch 2.17.

Backend

cd app/backend

# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your database credentials, OpenSearch host, etc.

# Run database migrations
alembic upgrade head

# Start the API server
uvicorn app.main:app --host 0.0.0.0 --port 8080

Frontend

cd app/frontend

# Install dependencies
npm install

# Development mode
npm run dev

# Production build
npm run build
# Serve dist/ with Nginx or any static file server

Verify Installation

# Health check
curl http://localhost:8080/ready
# Expected: {"status": "ok"}

# API docs (Swagger UI)
curl -s http://localhost:8080/docs | head -1

# OpenSearch connectivity
curl http://localhost:9200/_cluster/health?pretty

# PostgreSQL connectivity
docker compose exec postgres pg_isready -U mxtac
# Expected: accepting connections

# Check all containers are healthy
docker compose ps

Authentication

MxTac supports two authentication methods:

JWT Bearer Tokens (Web UI)

# Login to get tokens
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "analyst@mxtac.local", "password": "mxtac2026"}'

# Response: {"access_token": "eyJ...", "refresh_token": "eyJ...", "token_type": "bearer"}

# Use access token for API calls
curl http://localhost:8080/api/v1/overview/kpis \
  -H "Authorization: Bearer eyJ..."

API Keys (Automation)

# Generate an API key via the Admin panel or API
curl http://localhost:8080/api/v1/detections \
  -H "X-API-Key: mx_key_..."

Token Lifetime Configuration

ACCESS_TOKEN_EXPIRE_MINUTESShort-lived (default: 60 min). Used for API requests.
REFRESH_TOKEN_EXPIRE_DAYSLong-lived (default: 7 days). Used to rotate access tokens.

Database (PostgreSQL)

PostgreSQL 16 stores user accounts, rule metadata, connector configurations, and alert state.

# Connection string format
DATABASE_URL=postgresql+asyncpg://user:password@host:5432/database

# Run migrations
cd app/backend
alembic upgrade head

# Check migration status
alembic current

# Create a new migration
alembic revision --autogenerate -m "description"
Tip
In Docker Compose, migrations run automatically on container start. For manual installs, run alembic upgrade head after every update.

OpenSearch

OpenSearch 2.17 serves as the event store for normalized security events (OCSF schema).

# Basic (no auth)

OPENSEARCH_PORT=9200

# With security enabled
OPENSEARCH_HOST=opensearch.internal
OPENSEARCH_PORT=9200

OPENSEARCH_PASSWORD=StrongP@ss
OPENSEARCH_USE_SSL=true

Performance Tuning

# docker-compose.yml — increase JVM heap for production
environment:
  - "OPENSEARCH_JAVA_OPTS=-Xms2g -Xmx2g"  # 50% of available RAM, max 32g

Index Lifecycle

MxTac creates daily indices with the pattern mxtac-events-YYYY.MM.DD. Configure retention with ISM (Index State Management) policies:

# Example: Delete indices older than 90 days
curl -X PUT "http://localhost:9200/_plugins/_ism/policies/mxtac-retention" \
  -H "Content-Type: application/json" \
  -d '{
    "policy": {
      "description": "MxTac event retention",
      "states": [
        { "name": "hot", "actions": [], "transitions": [
          { "state_name": "delete", "conditions": { "min_index_age": "90d" } }
        ]},
        { "name": "delete", "actions": [{ "delete": {} }] }
      ]
    }
  }'

TLS / HTTPS

In production, terminate TLS at the Nginx reverse proxy.

# nginx/nginx.conf (minimal example)
server {
    listen 443 ssl http2;
    server_name mxtac.example.com;

    ssl_certificate     /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;

    # Frontend (static files)
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    # Backend API proxy
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket proxy
    location /ws {
        proxy_pass http://backend:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

# HTTP redirect
server {
    listen 80;
    server_name mxtac.example.com;
    return 301 https://$host$request_uri;
}

Agents

Agent Overview

MxTac ships two lightweight detection agents, both written in Rust for minimal resource footprint. They output native OCSF events directly — no conversion layer required.

AgentTypeRAMCPUBinary Size
MxGuardEDR (Endpoint)10-30 MB0.5-1%Single static binary
MxWatchNDR (Network)40-120 MB1-3% per coreSingle static binary

MxGuard (Endpoint Detection & Response)

MxGuard monitors file integrity, processes, network connections, authentication, registry, and scheduled tasks.

Installation

# Download the latest release (Linux x86_64)
curl -LO https://github.com/fankh/new-research/releases/latest/download/mxguard-linux-amd64
chmod +x mxguard-linux-amd64
sudo mv mxguard-linux-amd64 /usr/local/bin/mxguard

# Or build from source
cd agents/mxguard
cargo build --release
sudo cp target/release/mxguard /usr/local/bin/

Configuration

# /etc/mxguard/config.toml
[server]
mxtac_url = "https://mxtac.example.com"
api_key   = "mx_key_..."
interval  = 30   # seconds between collection cycles

[modules]
file_integrity  = true     # T1105, T1547, T1574, T1036
process_monitor = true     # T1059, T1106, T1055, T1003
network_monitor = true     # T1071, T1095, T1021
auth_monitor    = true     # T1078, T1110, T1136
registry_watch  = true     # T1547, T1112, T1574 (Windows only)
schtask_watch   = true     # T1053

[file_integrity]
watch_paths = ["/etc", "/usr/bin", "/usr/sbin"]
exclude     = ["*.log", "*.tmp"]

Run as a systemd service

# /etc/systemd/system/mxguard.service
[Unit]
Description=MxGuard EDR Agent
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/mxguard --config /etc/mxguard/config.toml
Restart=always
RestartSec=10
User=root

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now mxguard
sudo systemctl status mxguard

MxWatch (Network Detection & Response)

MxWatch captures and analyzes network traffic for C2 beacons, port scans, data exfiltration, and DNS tunneling.

Performance

PlatformCapture EngineThroughput
LinuxAF_PACKET + MMAP1-5M pps
Windows / macOSlibpcap~100K pps

Supported Protocols

HTTP/HTTPS DNS TLS/SSL SMB SSH FTP SMTP LDAP RDP DHCP NTP ICMP

Configuration

# /etc/mxwatch/config.toml
[server]
mxtac_url = "https://mxtac.example.com"
api_key   = "mx_key_..."

[capture]
interface = "eth0"
bpf_filter = "not port 22"    # Exclude SSH management traffic

[detection]
c2_beacon     = true    # Timing analysis for periodic callbacks
port_scan     = true    # SYN scan / connect scan detection
exfiltration  = true    # Large outbound data transfer alerts
dns_tunnel    = true    # High-entropy DNS query detection

Integrations

MxTac integrates with existing OSS security tools via connectors. Each connector normalizes events to the OCSF (Open Cybersecurity Schema Framework) standard.

Wazuh (HIDS / EDR)

Wazuh 4.7+ provides host-based intrusion detection, file integrity monitoring, and vulnerability assessment.

Connect to Wazuh

  1. Navigate to Settings > Integrations in the MxTac dashboard
  2. Click Add Integration > Wazuh
  3. Enter your Wazuh API credentials:

Zeek 6.x provides deep network traffic analysis, protocol parsing, and connection logging.

Connect to Zeek

Log Path/opt/zeek/logs/current/

Sigma Rules

MxTac executes Sigma rules natively via pySigma. No conversion to backend-specific query language needed.

Upload Rules

# Upload a single rule
curl -X POST http://localhost:8080/api/v1/rules \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@proc_creation_win_powershell_download.yml"

# Upload a directory of rules
for f in sigma_rules/*.yml; do
  curl -X POST http://localhost:8080/api/v1/rules \
    -H "Authorization: Bearer $TOKEN" \
    -F "file=@$f"
done

Rule Directory

Place Sigma rules in app/backend/sigma_rules/ for automatic loading at startup.

Troubleshooting

Common Issues

Backend won't start: "Connection refused" to PostgreSQL

# Ensure PostgreSQL is healthy
docker compose ps postgres
docker compose logs postgres

# Check database exists
docker compose exec postgres psql -U mxtac -c '\l'

OpenSearch: "max virtual memory areas vm.max_map_count [65530] is too low"

# Fix (temporary)
sudo sysctl -w vm.max_map_count=262144

# Fix (persistent)
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

Frontend can't reach backend API

# Check CORS_ORIGINS includes your frontend URL
# Check VITE_API_BASE_URL points to backend
# Check backend is healthy
curl http://localhost:8080/ready

High memory usage from OpenSearch

# Reduce JVM heap (default is 512m)
# In docker-compose.yml:
environment:
  - "OPENSEARCH_JAVA_OPTS=-Xms256m -Xmx256m"

Containers keep restarting

# Check logs for the failing container
docker compose logs --tail 50 backend

# Check health status
docker compose ps

# Rebuild images after code changes
docker compose build --no-cache backend

Database migration errors

# Check current migration state
docker compose exec backend alembic current

# Force migration to latest
docker compose exec backend alembic upgrade head

# Reset (destructive!)
docker compose exec backend alembic downgrade base
docker compose exec backend alembic upgrade head
